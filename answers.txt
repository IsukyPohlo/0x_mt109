	1. Las Unit Tests como su nombre dice, son para probar unidades funcionales de codigo. 
		Luego, las pruebas de integracion juntan a las unidades probadas anteiormente y comprueban su
		funcionamiento en conjunto y por ultimo las pruebas de aceptacion las realiza el cliente al que va dirigido el codigo
		para comprobar el funcionamiento correcto de todas las partes.
	
	2. Utiliza la metodologia Agil y se basa en realizar sprints que duran un determinado tiempo en el que se deben fijar objetivos
		a realizar. Despues de esto, todos los dias se revisa el progreso de todos en una reunion. Al terminar el sprint se pide
		retroalimentacion a los clientes.

	3. El TDD funciona primero creando pruebas unitarias en las que una vez que se cumplen, se debe refactorizar el codigo para mejorarlo.
		Es importante usar esta metofologia porque permite tener certeza del correcto funcionamiento de cada unidad de codigo, permite 
		supervisar y cuentizar el trabajo realizado para poner objetivos.

	4. El CI/CD permite facilitar el proceso de integracion del codigo que se refiere a la parte de la compilacion del programa, la construccion
		y el testeo. Despues de esto, la parte de la entrega (delivery) es poder desplegar el programa funcional.
		Esto, utilizando software de control de versiones y se busca automatizarlo para eficientar en tiempo y recursos.
	
	5. Las aplicaciones monoliticas incluyen todas las funciones dentro del mismo programa. Mientras que las aplicaciones basadas
		en microservicios dividen sus funcionalidades en distintas aplicaciones que no se comunican entre ellas.
